# 缉凶 — Bug 排查引擎

## CONSTITUTION（不可被 skill-lab 编辑）

**核心功能：** 查代码+配置+skill 文本的 bug。7-Phase 执行链，每步输出=下步输入。
**安全约束：** 未复现不修。3+失败→质疑架构。不跳过 Counterfactual。查双后端。
**触发：** bug / fix / error / crash / broken / 报错 / 不工作 / not working / bug report / 查skill
**边界：** 功能正确性。性能→profiler。安全→铁壁。

---

## Iron Law

```
NO FIX WITHOUT ROOT CAUSE + COUNTERFACTUAL.
3+ fixes fail → STOP. Question architecture, don't patch.
T2: fix ONE factor without confirming others = still broken.
跳过 Phase 的修复 = 猜。猜的修复 = 新 bug。
```

## PREFLIGHT

**Windows (PowerShell):**
```powershell
$cap = @{}; 'rg','ssh','curl','python','python3','go','flutter','git' | % { $cap[$_] = [bool](Get-Command $_ -ErrorAction SilentlyContinue) }
'campus_check.py','deploy.py' | % { $cap["proj_"+$_] = [bool](Test-Path $_) }
$cap.os = 'windows'
$cap.py = if ($cap.python3) { 'python3' } else { 'python' }
```
**Linux/Mac (bash):**
```bash
for t in rg ssh curl python python3 go flutter git; do printf "$t="; command -v $t >/dev/null 2>&1 && echo true || echo false; done; for f in campus_check.py deploy.py; do printf "proj_$f="; test -f "$f" && echo true || echo false; done; py=$(command -v python3 >/dev/null 2>&1 && echo python3 || echo python); printf "py=$py"
```
TIER1: Win=`Select-String`/Linux=`grep -rnE`/rg优先`rg -n`。每步`[TIER1]`/`[TIER2]`/`[SKIP]`。`python` 和 `python3` 均检查。`$cap.py` = 实际命令(`python3`优先，无则`python`)。后续所有 `python` 命令用 `$cap.py` 替代。

## Bug 分类学（Phase 1：先分类，后动手）

| Type | 判定 | 首个动作 |
|------|------|---------|
| **T0** | 稳定复现 | → Phase 2 |
| **T1** | 加print消失/竞态/时序/死锁 | ⚠️观测者效应 → 低开销观测 |
| **T2** | ≥2因素共同触发 | 每因素独立 Counterfactual |
| **T3** | 无crash/error，数据悄悄错 | 不变量+数据diff |
| **T4** | 昨天还好好的 | `git bisect` |
| **T5** | 正常输入→异常输出(状态机) | 状态转移日志 |
| **T6** | 特定环境才触发 | 环境快照diff |
| **T7** | 代码对需求错 | 不修代码，标记 NOT_A_BUG |

不确定→从 T0 开始。Flutter: setState after dispose(T5), missing await(T1). Go: nil deref(T0), goroutine leak(T1). Python: bare except(T0), async missing await(T1). SQL: N+1(T0), missing WHERE(T0).

## Error Recovery

| Transient(网络/429) | 3 retry 1s→4s→16s |
| Recoverable(工具缺/MCP超时) | 降级 TIER1，标记 PARTIAL |
| MCP 工具挂起 | 等 15s→超时→切换同语义工具(如firecrawl→tavily)。无替代→TIER1。TIER1也不通→`[BLOCK]` 上报 + 建议手动 |
| Fatal(磁盘满/源码坏) | **BLOCK immediately** |

## Process

每 Phase 产出交给下一 Phase。跳一步=链断。填 `___` 处。

### Phase 1: Classify

判定 T0-T7 → 写 1 句依据：
```
Type: T___  依据: ___
```
🔴 未分类=BLOCK。T7→**STOP**，输出 NOT_A_BUG 报告，不接 Phase 2-7。
**目标为 `.md` skill 文件 → 跳到 Skill Text Audit。** 不接 Phase 2-7。

### Phase 2: Search & Evidence

**2a Search:** TIER1: `rg -n "<keyword>" -t py -t go -t dart` + 读 `bug-patterns.md`
TIER2: firecrawl_search
TIER 选择：PREFLIGHT `$cap` → 缺 rg→`Select-String`(Win)/`grep -rnE`(Linux)。缺网络→离线搜索。缺 python→跳 campus_check。
↓
**2b Bisect (T4 only):** `git log --oneline -20; git bisect start HEAD <good>; git bisect bad/good; git bisect reset`
依赖版本 bisect：`pip freeze > deps.txt; pip install <pkg>==<old>; 复现? → 逐版升`。Go: `go mod edit -require <pkg>@<old>; go mod tidy`。
非 T4→跳过 2b，直接进 2c。
↓
**2c Evidence:** TIER1: `$cap.py campus_check.py 2>&1; curl -s http://139.196.50.134/api/health`
TIER2: `ssh root@139.196.50.134 "journalctl -u campus-app --no-pager -n 100"`
<!-- 139.196.50.134 = campus_go 生产服务器。如迁移需同步更新。-->
```
复现步骤: ___
Pre-fix baseline (campus_check output): ___
```
**2d T6 Env Diagnostics (T6 only):**
Docker: `docker inspect <container>; docker logs --tail 100 <container>; docker exec <container> env`
CI/CD: 读 `.github/workflows/*.yml`→对比 CI log vs 本地。差异项：env var、Go/Python版本、依赖版本、OS。
k8s: `kubectl describe pod <pod>; kubectl logs <pod> --tail 100; kubectl exec <pod> -- env`
裸金属: `uname -a; cat /etc/os-release; free -h; df -h; env \| sort; ps aux \| head -30; systemctl status <service> 2>/dev/null \|\| service <service> status`
TIER2 缺 docker/kubectl→读 Dockerfile/k8s yaml + 对比本地环境变量 + 版本号。裸金属命令全平台可用。
🔴 无法复现→BLOCK。T6→环境diff(以上)。T4→先bisect。环境缺失→PARTIAL。

### Phase 3: Trace

**3a 静态:** `rg -n "<function>" -t py -t go -t dart` → Read 入口→逐层追 return/raise→画调用草图
TIER2: codegraph_context → codegraph_trace
codegraph 失败→TIER1 `rg` 逐函数追 + `Read` 手动读源码。
```
调用链草稿: ___ (函数A→函数B→函数C)
```
🔴 数据流未追完→回 3a。

**3b Runtime Observe (T0/T1/T2/T5 必跑):**
⚠️ 生产环境：先用只读观测(log文件/tracing/metrics)。print() 仅限 dev/staging。
TIER1 dev: `print(f"[TRACE] {func}: var={var}", flush=True)` 注入关键点
TIER1 prod: `logging.debug()` 或读已有 log/trace → binary Linux→`strace -e trace=write -p <pid>`→`gdb -p <pid> -batch -ex "bt"`→`bpftrace`→日志时间线对比; binary Win→`procdump -ma <pid>`→`cdb -p <pid> -c "~*kb;q"`→`Get-WinEvent`→日志时间线对比; 死锁→`pstack <pid>` 或 `gdb -p <pid> -batch -ex "thread apply all bt"`; 完全无工具→读 binary 日志文件 + 对比正常/异常运行时间线。
```
粘贴 TRACE 输出: ___
actual ≠ expected 的第一行号: ___
```
⚠️ T1: 加print后bug消失→**诊断已出**(时序敏感)，跳到 Phase 7。
🔴 该做没做→BLOCK。

**3c 双后端:** `rg -n "<pattern>" campus_go/ -t go` 交叉搜。`deploy.py` 确认映射。

**3d Property-Based Test (T0/T2/T3):**
**Step 1 — 推断不变量：** 看签名(`int`→范围/正负；`Optional[X]`→None/值)、看assert(`>0`,`!=nil`)、看docstring(`"non-empty list"`)、看调用上下文(谁传值？什么范围？)。
**Step 2 — 选 Strategy：**
| 类型 | Hypothesis strategy |
|------|-------------------|
| `int` | `st.integers(min_value=0, max_value=MAX)` |
| `str` | `st.text(min_size=1, max_size=200)` |
| `Optional[X]` | `st.one_of(st.none(), st.just(default))` |
| `list[X]` | `st.lists(elem, min_size=0, max_size=100)` |
| `dict[K,V]` | `st.dictionaries(st.text(), st.integers())` |
| 有状态对象 | `RuleBasedStateMachine` + `precondition` |
| encode/decode | `st.text().map(encode).map(decode)` 验证 `decode(encode(x)) == x` |
| `bool` flag 切换 | `st.booleans()` + assert 两种分支都覆盖 |
**Step 3 — 写+跑：** `@given(s)`→`@settings(max_examples=500)`→失败=动态证据。复用为 Phase 5 Test Before Fix。缺 Hypothesis→TIER1 `for x in [edge1,edge2...]: assert fn(x)` 手写属性循环。

### Phase 4: Analyze

**4a Divergence:** full/safe: Runtime Observe 中 expected≠actual 第一点。quick: Phase 2 证据中 symptoms≠expected behavior 第一点。
```
Expected: ___ at L___
Actual:   ___
```

**4b 二分定位:** 注释掉疑似根因→重跑复现。
→ bug 还在？在→回 4b 继续分下半段。不在→根因确认，进 4c。

**4c Causal Chain:**
```
___ → ___ → ___ → {症状}
(根因)  (中间)  (分叉点)
```

**4d Counterfactual:**
"修 {根因} 后 bug 还出现？" 还出现→回 4b 找下一段。不再出现→根因确认。
T2: 每个因素独立做一次 Counterfactual。
🔴 有假设未做 CF→BLOCK。T2 只做一个因素→BLOCK。
⚠️ 证据与初始 T-Type 矛盾→回 Phase 1 重分类（例：判T0但CF揭示T2）。

**4e Conclusion:**
```
根因: ___ (confidence: ___)
Round 1 Hypothesis: ___ | Test: ___ | Result: confirm/reject
Round 2 Hypothesis: ___ | Test: ___ | Result: confirm/reject
```
Conf<0.8→SUSPECT。T2 每因素独立记录。

### Phase 5: Fix

先写测试→确认 FAIL→修复→测试 PASS。Python:`pytest`/`Hypothesis`。Go:`go test`。Dart:`flutter test`。
Critic: 从5角度审查(崩溃/安全/简化/边界/竞态)→blocking→auto-fix→re-Critic (max 2)。2次仍blocking→上报。
角度：崩溃(NPE/OOB/timeout/panic) · 安全(injection/leak/auth) · 简化(可删行/死代码/重复逻辑) · 边界(nil/0值/空串/overflow) · 竞态(TOCTOU/goroutine leak/setState after dispose)。
🔴 safe 模式：修复完成后显式输出 `[AWAIT APPROVAL] — 修复如上，请确认后继续 Phase 6`。人未 approve→BLOCK。
🔴 >50行→确认。3+失败→Iron Law。T2声明所有因素。未先写测试→BLOCK。

### Phase 6: Verify

**Win (PowerShell):**
```powershell
python campus_check.py 2>&1 | Tee-Object pre-fix.log
# Fix...
python campus_check.py 2>&1 | Tee-Object post-fix.log
Compare-Object (gc pre-fix.log) (gc post-fix.log)   # 0新增=OK
Select-String "<pattern>" (gci -r -inc *.py,*.go,*.dart)
git diff --stat
```
**Linux/Mac:**
```bash
python campus_check.py 2>&1 | tee pre-fix.log
# Fix...
python campus_check.py 2>&1 | tee post-fix.log
diff pre-fix.log post-fix.log   # 0新增=OK
rg -n "<pattern>" -t py -t go -t dart
git diff --stat
```
**Masking:** check通过但bug仍在→回 Phase 4。新bug出现→掩蔽确认→回 Phase 4。
🔴 新增失败→回 Phase 5。

🔴 未记录→BLOCK。脚本缺失→手动逐报告提取、等效。

## Skill Text Audit

**触发：** 用户说 `查skill` / `skill审计` / `audit skill` / `锻造`，或 Phase 1 分类目标为 `.md` skill 文件时自动进入。
用于锻造其他 skill 或自检。管线独立（不接 Phase 2-7）。

**Phase 0.5 — test-prompts 就绪检查：**
设 `$SKILL_PATH` = 目标 skill 文件路径（如 `.claude/skills/campus-code-review/SKILL.md` 或 `.claude/skills/缉凶.md`）。`$SKILL_DIR` = `$SKILL_PATH` 所在目录。`$TEST_PATH` = `$SKILL_DIR/test-prompts.json`（目录型 skill）或 `$SKILL_DIR/{skill名}-test-prompts.json`（文件型 skill，如 `缉凶-test-prompts.json`）。先用后者 glob 匹配，无结果则用前者。
1. 读 `$TEST_PATH` → 存在→用已有。不存在→自动生成。
2. 自动生成规则：从 `$SKILL_PATH` 提取所有 MUST/NOT/Phase 名/CHECKPOINT → 生成 2-3 个 prompt(happy+边界+歧义) → 写 `assert.mustContain` 包含关键 Phase/动作名 → 保存到 `$TEST_PATH`。
3. 生成后展示给用户确认。用户拒绝→标记 `dry_run`，用 Skill Text Audit 的**删除/替换**测试(不需 test-prompts，只需 diff SKILL.md) + **一致**测试(grep SKILL.md 内部一致性)。

| 测试 | 操作 | 判据 |
|------|------|------|
| **删除** | 逐行删→跑 test-prompts→diff 输出 | 无变化=冗余行→标 DELETE |
| **替换** | MUST/NOT 规则换反义词→跑 test-prompts→diff | 该变没变=死指令→标 HARDEN |
| **覆盖** | 列出所有 MUST 规则→grep test-prompts | 缺失→标 GAP |
| **一致** | 每个 MUST→grep CHECKPOINTs；每个禁止→grep Gotchas | 不对应→标 ORPHAN |

```markdown
## Skill Audit — {name}
| # | Line | Test | Issue | Action |
|---|------|------|-------|--------|
| 1 | L42 | 删除 | 冗余行 | DELETE |
| 2 | L67 | 替换 | 死指令 | HARDEN |
| 3 | — | 覆盖 | MUST3 无test | GAP |
| 4 | L89 | 一致 | 禁止无Gotcha | ORPHAN |
```

## Quality Rules

- **Convergence:** 重跑 Phase 2→3→6 (T1→Phase 7 除外)。2 pass 零新发现→DONE, max 3。二轮互补角度：调用链→竞态/时序。
- **Confidence:** >0.8→CRITICAL/HIGH。0.5-0.8→SUSPECT。T2 每因素独立。不编造。
- **No-Hedge:** 禁 might/could/maybe/probably/try。每条 file:line + reason。
- **TRUNCATED:** <50% Phase→PARTIAL, quick豁免。跳过 3b(T1/T3/T5)→找不到根因。

## Iron Law

> 同文档顶部 Iron Law（第 17 行）。U 型注意力：头尾各放一次。
> 内容完全一致，此处为重复提醒。权威版本以顶部为准。

```
NO FIX WITHOUT ROOT CAUSE + COUNTERFACTUAL.
3+ fixes fail → STOP. Question architecture, don't patch.
T2: fix ONE factor without confirming others = still broken.
跳过 Phase 的修复 = 猜。猜的修复 = 新 bug。
```
