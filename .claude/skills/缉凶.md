# 缉凶 v2.0 — 合同框架

## CONSTITUTION（不可被 skill-lab 编辑）

**核心功能：** 查代码+配置+skill 文本的 bug。合同链强制执行：每步产出=下步门票。
**安全约束：** 不复现不修。不 Counterfactual 不提交。不测试不修。3+失败→质疑架构。
**触发：** bug / fix / error / crash / broken / 报错 / 不工作 / not working / bug report / 查skill
**边界：** 功能正确性→缉凶。性能→profiler。安全→铁壁。

---

## Red Lines（违反任一条 = 修复无效）

| # | 红线 | 违反后果 |
|---|------|---------|
| 1 | **不复现不修** — 至少 1 次稳定复现 + 记录复现步骤 | 根因是猜的 |
| 2 | **不 Counterfactual 不提交** — "修了 X 后 bug 不再出现"→确认 | 根因未确认 |
| 3 | **不测试不修** — 先写能复现 bug 的测试→FAIL→修复→PASS | 修复不可验证 |
| 4 | **3 次修不好 → STOP** — 不是技术问题，是理解问题。重读证据 | 继续改=新 bug |
| 5 | **修前确认部署文件** — `deploy.py` 决定哪个文件跑在生产上 | 修错文件 |

---

## 合同链（每步产出 = 下一步门票）

```
[分类] → [证据] → [追踪] → [分析] → [修复] → [验证] → [记录]

规则：上一步产出填完整 → 下一步才能开始。
      任一步产出为空 → 后续全部无效。
      事后补填 → 标记 POST-HOC，置信度降一级。
```

| 步 | 产出（必须填写） | 未填写的后果 |
|----|----------------|-------------|
| **分类** | `Type: T___ 依据: ___` | 后续搜索无目标 |
| **证据** | `复现步骤: ___` + baseline 输出 | 追踪无起点 |
| **追踪** | `调用链: ___` + `Expected vs Actual 第一点: ___` | 分析无数据 |
| **分析** | `根因: ___ (conf: ___)` + Counterfactual 结果 | 修复无依据 |
| **修复** | 测试 FAIL→PASS 完整输出 | 验证无基线 |
| **验证** | pre/post diff + `git diff --stat` | 记录不完整 |
| **记录** | `.fixes/{date}-{slug}.md` + bug-patterns 更新 | 经验丢失 |

## Bug 分类

| Type | 判定 | 首个动作 |
|------|------|---------|
| **T0** | 稳定复现 | 标准合同链 |
| **T1** | 加 print 消失/竞态/时序 | ⚠️观测者效应。print 仅 dev。prod 用只读观测 |
| **T2** | ≥2 因素共触 | 每因素独立 CF。修一个不收工 |
| **T3** | 无报错数据错 | 不变量+数据 diff + Property-Based Test |
| **T4** | 昨天还好 | git bisect + 依赖版本 bisect |
| **T5** | 状态机异常 | 状态转移日志 |
| **T6** | 特定环境 | 环境快照 diff（Docker/CI/k8s/裸金属） |
| **T7** | 代码对需求错 | **STOP**。NOT_A_BUG。不修代码 |
| **Skill** | 目标为 `.md` 文件 | → Skill Text Audit（独立管线） |

不确定→从 T0 开始。

## MEL 模式

| 模式 | 合同链 | 触发 |
|------|--------|------|
| **full** | 7 步全走 | 默认 |
| **quick** | 分类+证据+分析（不修） | 快速/诊断/看看 |
| **safe** | 7 步 + Phase 5 [AWAIT APPROVAL] | 重要修复 |

|---------|------|

## Worked Example

```
Bug: 活动列表 API 返回 500 (main_remote:222)
→ 分类: T0 (每请求必 500)
→ 证据: curl /api/activities → 500; campus_check baseline captured
→ 追踪: codegraph_trace GetActivities→ListActivities→countByStatus
         Expected: rows[].status at L67. Actual: panic nil deref (rows=nil)
→ 分析: DB timeout→nil rows→未 nil-check→panic→500
         Counterfactual: 加 nil-check→curl→200 OK。CF 通过。
         Root: nil-check 缺失 (conf:0.95)
→ 修复: go test TestGetActivitiesNilRows→FAIL→加 if rows==nil→PASS
→ 验证: pre/post diff 0 新增。rg rows[ →无其他无保护访问
→ 记录: .fixes/20260621-nil-rows-panic.md + bug-patterns 追加
```
