# 铁壁 — 全栈安全审计

## CONSTITUTION（不可被 skill-lab 编辑）

**核心功能：** 代码安全审计 → 结构化报告。7步：密钥扫描→SAST→端点审计→硬编码密钥→依赖CVE→服务器扫描→DB审计。
**安全约束：** 绝不编造finding。绝不凭记忆审——先读文件。绝不跳过Go后端。`.audits/` 审计产物不入源码（应在 `.gitignore`）。
**触发：** 铁壁 / audit security / check security / 安全审计 / 安全扫描 / quick audit / 快速扫描 / check secrets / 查密钥 / check dependencies / 查依赖 / gitleaks / semgrep

---

## PREFLIGHT（每次必跑，零依赖）

```powershell
# 检测可用工具 + OS — PowerShell try/catch不捕获外部命令"未找到"错误，必须用 Get-Command
$tools = @{}
$tools.gitleaks = [bool](Get-Command gitleaks -ErrorAction SilentlyContinue)
$tools.semgrep = [bool](Get-Command semgrep -ErrorAction SilentlyContinue)
$tools.rg = [bool](Get-Command rg -ErrorAction SilentlyContinue)
$tools.pipaudit = [bool](Get-Command pip-audit -ErrorAction SilentlyContinue)
$tools.nuclei = [bool](Get-Command nuclei -ErrorAction SilentlyContinue)
$tools.os = if ($IsLinux) { 'linux' } elseif ($IsMacOS) { 'mac' } else { 'windows' }
Write-Host "CAPABILITY: $($tools | ConvertTo-Json -Compress)"
```
TIER1 命令自动适配平台：
- **Windows:** `Get-ChildItem -Recurse ... | Select-String ...`
- **Linux/Mac:** `grep -rnE 'pattern' --include='*.py' --include='*.go' .`
- **通用（rg可用时优先）：** `rg -nE 'pattern' -t py -t go`

每步标注来源层级：`[TIER1]` = 零依赖原生命令 / `[TIER2]` = 外部工具 / `[SKIP]` = 两层都不可用则跳过并标记 PARTIAL。

## Step 1 — 密钥扫描

**TIER1（零依赖，永远可用）：**
```powershell
# Windows: Select-String with ErrorAction on binary files
Get-ChildItem -Recurse -Include *.py,*.dart,*.go,*.yaml,*.toml,*.sh,*.js,*.ts -ErrorAction SilentlyContinue |
  Select-String -Pattern '(password|secret|key|token|api_key|AKIA|ghp_|-----BEGIN)\s*[=:]\s*["''][^"'']{6,}' -ErrorAction SilentlyContinue |
  ForEach-Object { "$($_.Path):$($_.LineNumber): $($_.Line.Trim())" }
```
```bash
# Linux/Mac: grep with common secret patterns
grep -rnE '(password|secret|key|token|api_key|AKIA[[:alnum:]]{16}|ghp_[[:alnum:]]{36}|-----BEGIN) *[=:] *["'"'"'][^"'"'"']{6,}' \
  --include='*.py' --include='*.dart' --include='*.go' --include='*.yaml' --include='*.toml' --include='*.sh' --include='*.js' --include='*.ts' .
```
**TIER2（可选升级）：** `gitleaks detect --source . --no-git --verbose` — 精度更高，覆盖更多模式

先读 `.gitleaks.toml` allowlist 再报。pet_config.json 和 `*.log` 在 allowlist 中。

**门控：** 发现真实生产密钥（非 test/example/allowlisted）→ BLOCK immediately，跳到 Step 8。仅误报 → PASS。

## Step 2 — SAST

**TIER1（零依赖）：**
```powershell
# Windows
Get-ChildItem -Recurse -Include *.py -ErrorAction SilentlyContinue | Select-String -Pattern 'except\s*(Exception)?\s*:\s*pass' -ErrorAction SilentlyContinue | ForEach-Object { "$($_.Path):$($_.LineNumber): bare except:pass" }
Get-ChildItem -Recurse -Include *.py -ErrorAction SilentlyContinue | Select-String -Pattern 'SELECT \*' -ErrorAction SilentlyContinue | ForEach-Object { "$($_.Path):$($_.LineNumber): SELECT *" }
Get-ChildItem -Recurse -Include *.py,*.go,*.dart -ErrorAction SilentlyContinue | Select-String -Pattern 'http://' -ErrorAction SilentlyContinue | ForEach-Object { "$($_.Path):$($_.LineNumber): http:// URL" }
Get-ChildItem -Recurse -Include *.py -ErrorAction SilentlyContinue | Select-String -Pattern '(execute|executemany)\(.*\%|format.*\)' -ErrorAction SilentlyContinue | ForEach-Object { "$($_.Path):$($_.LineNumber): potential SQL injection" }
Get-ChildItem -Recurse -Include *.py -ErrorAction SilentlyContinue | Select-String -Pattern 'subprocess\.(call|Popen|run)\(.*shell\s*=\s*True' -ErrorAction SilentlyContinue | ForEach-Object { "$($_.Path):$($_.LineNumber): shell=True injection risk" }
```
```bash
# Linux/Mac
grep -rnE 'except\s*(Exception)?\s*:\s*pass' --include='*.py' .
grep -rnE 'SELECT \*' --include='*.py' .
grep -rnE 'http://' --include='*.py' --include='*.go' --include='*.dart' .
grep -rnE '(execute|executemany)\(.*(%)|\.format.*\)' --include='*.py' .
grep -rnE 'subprocess\.(call|Popen|run)\(.*shell\s*=\s*True' --include='*.py' .
```
**TIER2（可选升级）：** `semgrep --config="f:/ClaudeFiles/.semgrep" --error`

Go/Dart后端补充手动审查：未检查的 `err != nil` 返回、SQL字符串拼接、defer 泄漏。

**门控：** ERROR 发现 → 标记 BLOCKING，继续 Step 3。仅 WARNING → PASS。

## Step 3 — 端点安全审计

对每个端点类别执行半形式推理（Premise→Trace→Check→Conclusion），不确定的标记 ⚪ SUSPECT（不猜 CRITICAL）。

**端点类别检查项：**
- **Auth:** JWT 拒绝 alg:none？is_active 校验？401 非 403？密码强度？MFA？
- **Write endpoints:** 权限校验？FOR UPDATE 防竞态？输入验证（类型/长度/范围）？
- **Read endpoints:** 禁止 SELECT *？隐私字段受保护？分页上限？
- **Admin endpoints:** 仅超管可调？操作日志记录？
- **Upload:** MIME 验证（非扩展名）。大小限制。UUID 文件名。auth 后才可访问。
- **Notification/Message:** 发送者验证？内容过滤（冒充前缀/裸URL/XSS）？频率限制？

无 `.py`/`.go` 文件 → 跳过并标记 PARTIAL。

🔴 **CHECKPOINT Step 3:** 发现 CRITICAL auth bypass/SQLi → BLOCK immediately。仅 HIGH/MEDIUM → PASS 继续。SUSPECT → 标记并继续。

## Step 5 — 依赖 CVE

**预检：** `requirements.txt`/`pyproject.toml`/`go.mod`/`pubspec.lock` 是否存在？不存在 → 跳过。

**TIER1：** `pip list --outdated --format=json` → 对过期包手动查 cve.mitre.org / osv.dev
**TIER2：** `pip-audit`（更精确）。Go: `govulncheck ./...`。Dart: `dart pub deps --json`

注意：过期 ≠ 有 CVE。仅标记确认有 CVE 的依赖。

🔴 **CHECKPOINT Step 5:** CRITICAL CVE（RCE/权限提升，直接依赖）→ BLOCK immediately。仅 HIGH/MEDIUM → PASS 继续。无依赖文件 → SKIP。

## Step 7 — DB 审计

**TIER2（pg-ops或直连SQL）：**
```sql
SELECT query, calls, mean_exec_time FROM pg_stat_statements ORDER BY mean_exec_time DESC LIMIT 20;
SELECT pid, usename, state, query, wait_event FROM pg_stat_activity WHERE state != 'idle' AND wait_event IS NOT NULL;
SELECT schemaname, tablename, seq_scan, idx_scan FROM pg_stat_user_tables WHERE seq_scan > 10 ORDER BY seq_scan DESC;
```
pg-ops不可用→上述SQL直连。SQLite→跳过。无DB连接→跳过并标记PARTIAL。

## Step 8 — 结构化报告

```markdown

**Passes: N** | **Status: COMPLETE / PARTIAL — n/7 steps** | **Mode: TIER1/TIER2**

| Step | Source | Verdict | Findings |
|------|--------|---------|----------|
| 1 Secret Scan | {TIER1/TIER2} | PASS/FAIL | {n} findings |
| 2 SAST | {TIER1/TIER2} | PASS/FAIL | {n} findings |
| 3 Endpoint Audit | — | PASS/FAIL | {n}/{total} endpoints |
| 4 Hardcoded Secrets | {TIER1/TIER2} | PASS/FAIL | {n} secrets |
| 5 Dependencies | {TIER1/TIER2} | PASS/FAIL | {n} CVEs |
| 6 Nuclei | {TIER2/SKIP} | PASS/FAIL/SKIP | {n} findings |
| 7 DB Audit | {TIER2/SKIP} | PASS/FAIL/SKIP | {n} issues |

### Overall Verdict: PASS / REVIEW NEEDED / BLOCKING
```

每条 CRITICAL/HIGH finding 必须有 file:line + concrete fix。内容先出 CRITICAL/HIGH（>5条发现或 >1000行 → 先问再展开）。不确定的标记 ⚪ SUSPECT。

收敛：完成后重扫有发现的步骤。两轮零新发现 → DONE。最多 3 轮。

产物写入 `.audits/{scope}-{YYMMDD}-{HHMM}.md`。同漏洞连续 2 次出现 → severity +1 升级。

## Gotchas（真实踩坑 — 运行前对照）

| # | 别做 | 替代 |
|---|------|------|
| 1 | 跳过 gitleaks/Step1 | 每轮必跑。TIER1 Select-String 永远可用 |
| 2 | 凭记忆审 allowlist | 先读 `.gitleaks.toml` 再报 |
| 3 | 忽略 WARNING | semgrep WARNING 可能掩盖真 bug |
| 4 | 只看 Python | Dart/Go/YAML/Shell 同样含密钥 |
| 5 | 跳过 Go 后端 | 未部署 ≠ 无 bug |
| 6 | 编造 finding 凑数 | 不确定 → SUSPECT，不编 CRITICAL |
| 7 | hedging 措辞（might/could/possibly） | 每条 finding 必须有 file:line + concrete fix |
| 8 | 跳过收敛重扫 | 至少 1 次 re-sweep |
| 9 | 不经确认跑 nuclei | Step 6 前必须用户确认 |
| 10 | 静默跳过工具失败 | 标记 PARTIAL + 说明哪个工具不可用 |
