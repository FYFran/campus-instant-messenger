# Bug分类详解（缉凶 reference）

## 决策流

1. 代码没改昨天能用？→T4。立刻查git log+配置变更
2. 每次必现crash/panic/500/nil deref/SIGSEGV→T0。必须有崩溃证据(stack trace/panic log/exit code≠0)。**判别铁律：看用户症状（用户看到crash/500了吗？），不看代码内部。代码nil+用户无crash→不是T0**
3. 同一操作有时正常有时异常？→T1。同一输入不同时间=不同结果。时序/竞态/Heisenbug
4. 换一个参数值就正常？→T2。不同输入=不同结果。区分:改输入能改变结果吗？能→T2，不能→T1
5. 正常输入→卡在中间状态回不来？→T5。状态机卡住。查状态转移日志
6. 1-5全排除了→T3。跳过1-5直接选T3=整条链报废。T3要求每次结果一致地错
7. 只在本机/CI/Docker/某OS出现？→T6。环境快照对比
8. 读代码发现行为符合产品设计？→T7。STOP NOT_A_BUG
9. 症状可被部署配置差异解释且近期有部署/重启？→T4(NOT代码bug)。T4专属配置检查通过前不许读源码

## Type速查

| Type | 症状 | 第一步 | 常见误判 |
|------|------|--------|---------|
| T0 | "每次都"、"必现"、"稳定复现" | 标准合同链 | — |
| T1 | "偶尔"、"有时候"、"刷新又好了"、"并发" | 只读观测不加print，时序分析 | 误判T0(间歇≠稳定)或T3(间歇也"无报错") |
| T2 | "同时满足"、"和...有关"、"某些...某些不" | 每因素独立CF | 误判T0或T3(不同输入不同结果) |
| T3 | "没报错"、"数字不对"、"越来越慢" | 不变量检查+数据diff | 先排T1/T2/T5再判T3 |
| T4 | "昨天还好"、"代码没改"、"重启后" | git bisect+配置diff+时间线 | 误判为代码bug |
| T5 | "卡住了"、"状态不变"、"一直pending" | 状态转移日志+状态机图 | 误判T0或T3(卡住≠crash) |
| T6 | "本机正常CI挂了"、"Win行Linux不行" | 环境快照diff | 误判T0 |
| T7 | "以为"、"应该是"、"安全审查说" | STOP。验证声称的问题是否真实存在 | 误判T0-T6 |

## 致命误判模式

| # | 模式 | 症状信号 | 真实Type |
|---|------|---------|----------|
| F1 | "偶尔不生效"+"刷新又好了" | 同一操作结果不一致 | T1(不是T3) |
| F2 | "某些学院名能看到，某些不能" | 不同输入不同结果 | T2(不是T3) |
| F3 | "一直pending不变" | 卡在中间状态 | T5(不是T3) |
| F4 | "偶尔扣钱不给结果"+"约20次1次" | 间歇但非竞态，逻辑错误间歇触发 | T3(不是T1) |
| F5 | "安全审查说XX不安全" | 第三方报告声称有bug | T7 |
| F6 | "昨天还好+重启后+代码没改"→读源码 | Agent立即读源码不查配置 | T4 |
| F7 | "T4+token/401/认证"→读auth.go | 不先curl直连后端端口测网络 | T4 |
| F8 | 找到意外bug但分类的是意外发现 | 症状锚定检查没做 | 原症状真实Type |
| F9 | T6→找代码修复不查环境差异 | 症状含版本/OS差异但追到代码bug就停 | 环境版本差异 |

## T4强制配置检查（3步走完才能读源码）

1. 配置diff: nginx.conf/docker-compose*.yml/.env/systemd unit/部署脚本—与git HEAD对比
2. 端口/服务检查: 确认服务监听端口和proxy_pass/upstream目标是否匹配
3. 启动日志: 检查上次重启/部署日志，找config loaded/reverted/warning/error

## T6强制环境检查（4步走完才能读源码）

1. 环境快照diff: go version/OS/Docker image tag/关键依赖版本—逐项对比
2. 行为差异复现: 在差异环境各跑一次相同输入
3. 变更记录: git log查CI配置/Dockerfile/go.mod/go.sum近期变更
4. ⚠ 版本差异存在→搜changelog/release notes中与失败组件相关的breaking changes

## 结构性排除（填完才能进证据步）

```
最可能的替代分类: T_?
替代分类的证据: ___（引用bug描述原文）
你的分类vs替代的决定性差异: ___（可验证的事实差异，不是"I think"）
```
