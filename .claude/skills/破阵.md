# Red Team v2.2 — 通用对抗演练

## CONSTITUTION（不可被 forge 编辑）

**核心功能：** 全对抗演练：3攻击者角色×7阶段×真实利用→攻击链报告+防御建议。红队发现→反馈铁壁+狩猎规则。
**Iron Law：** NO VULNERABILITY CLAIM WITHOUT REPRODUCIBLE EXPLOIT PATH.
**红线：** 绝不执行破坏性命令(DROP/DELETE/TRUNCATE生产)。绝不跳过"不可利用"文档。绝不只测nginx层。绝不假设后端安全。
**触发：** attack / red team / pentest / 渗透测试 / 安全测试 / exploit
**边界：** 对抗验证→破阵。安全审计→铁壁。漏洞修复→缉凶。
**模型：** deepseek-v4-pro。换模型→重跑BugHuntBench quick。

## Gotchas

| # | 症状 | 根因 | 教训 |
|---|------|------|------|
| 1 | 单独测每个漏洞=LOW | 两LOW串联=HIGH | 必须串漏洞成攻击链 |
| 2 | 扫nginx=应用层安全 | nginx是外层 | 绕过nginx直连后端端口 |
| 3 | "不可利用"没记录 | 以为没用 | 不可利用=防御有效→反馈防御模式 |
| 4 | 红队报告写完=完事 | 没反馈给铁壁 | 红队发现→更新SAST规则+狩猎指标 |
| 5 | 不可利用=白干了 | 没记录防御模式 | 不可利用=防御有效。输出NON_EXPLOITABLE模板：漏洞→防御→反馈铁壁→作为有效pattern |

## 攻击面枚举清单（每次演练第一步）

逐项检查，不可跳过：
- 路由注册: 公开vs保护组？WebSocket/上传/静态文件在JWT外？
- 认证: JWT alg=none？弱密钥？token在URL参数？
- 授权: college scope隔离？role检查在每个handler？
- 注入: SQL拼接字符串？XSS存储/反射？命令注入shell=True？
- 重放: 写操作有idempotency key？UNIQUE约束？nonce？
- 信息泄露: 响应含内部路径/栈追踪？role/id返回给非本人？
- 多后端: Go和Python handler行为不一致？

## 3角色×7阶段

阶段: 侦察→武器化→投放→利用→安装→C2→行动

## Finding 严重度分级

| 级别 | 定义 | 示例 |
|------|------|------|
| 🔴 CRITICAL | 认证绕过/RCE/数据泄露 | JWT伪造、SQLi RCE、未认证WebSocket推送敏感数据 |
| 🟠 HIGH | 权限提升/注入/攻击链 | college→school提升、XSS、信息泄露串联利用 |
| 🟡 MEDIUM | 单点弱点/配置问题 | 限流缺失、信息泄露(孤立/不可串联) |
| 🟢 LOW | 理论风险/不可利用 | "如果攻击者已经在内网"、"如果已经有了admin权限" |

**不可利用标记:** 发现防御有效→输出 `NON_EXPLOITABLE: {what} — {why blocked}` → 反馈铁壁作为有效防御模式

每次演练完成后：
```

2. 有没有攻击链出乎意料？→ 追加链模式
3. 防御建议是否被铁壁采纳？→ 追踪闭环
4. 假阳性有没有标记？→ 更新筛选规则
→ .fixes/{date}-redteam-pattern.md
→ 轮回采集→确认→注入
```
