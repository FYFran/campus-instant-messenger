## 关键发现

### 1. 缉凶 v2.0 合同链执行: 100% 完整
所有 10 个 bug 的排查都严格执行了 7 步合同链。没有跳过 gate 的情况。

### 2. 低分 ≠ 无能
B03/B04/B05 的低分源于 agent 在 bug description 模糊时，找到了代码中**真实存在的其他 bug**:
- B03: 找到后端 6 个端点缺 college 授权（真 security bug）
- B04: 找到 participant_hours NULL 传播导致 SUM 忽略（真 data bug）
- B05: 找到 Login token 更新非原子（真 race condition）

**这些是真发现，不是幻觉。BugHuntBench 的 bug description 需要更精确。**

### 3. BugHuntBench 改进方向
- Bug description 需要包含更具体的 scope（"后端代码问题" vs "nginx 配置问题"）
- 多 bug 共存时，需要区分"主要 bug"和"次要发现"
- 考虑加入"agent 发现了 ground truth 之外的 bug"作为加分项

## 执行方式

每个 agent 配备: 缉凶 v2.0 合同链 + campus_go 代码访问权
评分: 人工对照 ground truth 逐项判定
