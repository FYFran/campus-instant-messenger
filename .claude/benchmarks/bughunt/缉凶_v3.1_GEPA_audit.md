## 逐个 Transcript 分析

### B03 (2/8) — FALSE LOW: Agent 发现真 bug，基准不给分

**Agent transcript:**
```
Type: T_AUTH — 后端授权缺失
分类: T_AUTH 依据: 6个端点只检查role不检查college
证据: ApproveActivity等6端点activities_admin.go:117,153,200只检查role
追踪: POST /api/activities/:id/approve -> role check only -> no college validation
分析: 根因: Missing server-side college authorization (conf:0.90)
修复: Add college scope validation in all 6 endpoints
验证: college_admin_A -> 403 for college_B activities
```

**GT:** T0 — ListActivities SQL WHERE 子句缺少 `college = (SELECT college FROM users WHERE id=$userID)` 过滤

**判定:** Agent 发现的是**真实的 college 权限隔离漏洞**，在 admin 端点（ApproveActivity/RejectActivity等），不是在 ListActivities。两类问题同根同源——都是 college_admin 跨学院访问。Agent 的调查报告质量高、证据充分、有 CF。

**基准问题:** 评分系统只匹配 GT 文本，不判断"是否发现了同类的真 bug"。需要引入 partial-credit 机制。

**建议:** B03 重分类为 "valid_alternative_finding"，给予 partial credit (4-5/8)。
