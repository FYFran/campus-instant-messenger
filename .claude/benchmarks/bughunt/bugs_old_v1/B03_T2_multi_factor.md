# B03 — T2: multi-factor permission bypass

## Bug 描述
campus_go 中，college_admin 角色的用户有时能看到并操作其他学院的活动。权限系统设计是"college_admin 只能管自己学院"，但偶尔跨学院操作成功了。似乎跟活动所属学院名称包含特殊字符或跟管理员学院名称部分匹配有关。

## Ground Truth

**Type:** T2（≥2 因素共同触发 — 需要同时满足：学院名包含逗号 + 活动学院名是管理员学院名的子串）

**根因:** 
1. 权限检查使用 `strings.Contains()` 做学院匹配，"计算机" 匹配 "计算机科学" → true（误匹配）
2. 学院名字段使用逗号分隔存储多个学院，"计算机科学,电子工程" 中 "计算机" in split结果 因为空格问题 → false（漏匹配）
两个因素共同导致：有些 admin 被错误授权（因素1），有些被错误拒绝（因素2）。

**正确修复:** 
1. 改用精确匹配（`==` 或标准化后的集合查找）
2. 学院名存储改为 JSON 数组或关联表

**评分要点:**
- 分类: T2 — 识别多因素 (1pt)
- 证据: 两个独立复现条件 (1pt)
- 根因: strings.Contains + 逗号分隔 → 两个因素 (2pt)
- CF: 每个因素独立验证 (1pt)
- 修复: 精确匹配 + 数据结构修复 (1pt)
- 链完整 (1pt)
