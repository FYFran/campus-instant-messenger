## Step 2: SAST — 1 CRITICAL finding

### 🔴 CRITICAL: 存储型 XSS (S02)
- **位置:** `activities_admin.go:96-115` `CreateActivity`
- **根因:** `req.Description` 直接 INSERT 到数据库，未经任何 HTML 清洗
- **攻击场景:** 攻击者创建活动，description 包含 `<script>alert(document.cookie)</script>`，其他用户查看活动详情时脚本执行
- **修复:** 存储前用 `html.EscapeString()` 清洗，或引入 `bluemonday` sanitizer
- **OWASP:** A03:2021 Injection
- **代码证据:**
```
// activities_admin.go:96-115
`INSERT INTO activities (title, description, ...)
 VALUES ($1,$2,...)`,  // req.Description → $2, no sanitization
```
- **验证:** `grep -ri "html.EscapeString\|bluemonday\|sanitiz" campus_go/` → 0 results

### OK: JWT 算法验证
- `auth.go:95`: `if _, ok := t.Method.(*jwt.SigningMethodHMAC); !ok` — 拒绝非HMAC算法 ✅

### OK: 密码哈希
- `auth.go:124-131`: bcrypt + argon2id 双支持 ✅
- Register 使用 `bcrypt.DefaultCost` ✅

### OK: DB角色验证
- `auth.go:125-130`: JWT中间件从DB读取role，不信任token中的role — 防止角色提升 ✅

### 🟡 MEDIUM: RefreshToken 无限流
- **位置:** `main.go:46` `api.POST("/token/refresh", ...)`
- **问题:** 无任何速率限制，攻击者可无限刷新token
- **修复:** 添加 `middleware.RateLimit(10, time.Minute)`

### 🟡 LOW: Login限流用内存map
- **位置:** `auth.go:22-23` `var loginRateLimit = make(map[string]time.Time)`
- **问题:** 服务重启后计数器清零，攻击者可重启后继续暴力破解
- **修复:** 使用Redis或其他持久化计数器
