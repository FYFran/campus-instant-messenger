## R01: WebSocket 端点无认证 (R0) — 发现 ✅

**严重度:** 🟠 HIGH
**分类:** R0 认证绕过

**证据:**
- `main.go:75`: `api.GET("/ws", handlers.HandleWS)` 注册在公开 `api` 组
- 对比：所有数据端点（me/activities/dashboard）都在 `protected` 组（需 JWT + RateLimit）
- WebSocket 连接无需 Authorization header

**利用路径:**
```
1. 匿名客户端建立 WebSocket 连接 → 101 Switching Protocols
2. 接收实时推送（活动创建/公告发布/通知）
3. 信息收集 → 了解活动发布时间 → 精准抢活动名额
```

**根因:** 路由注册用了 `api.GET` 而非 `protected.GET`。应为注册到需 JWT 验证的 protected 组。

**修复:** 改为 `protected.GET("/ws", handlers.HandleWS)` 或在 HandleWS 中加 token 验证。

**严重度:** N/A
**分类:** Go vs Python 后端差异
